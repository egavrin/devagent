# packages/arkts

This file provides guidance to AI agents working in this component.

## Purpose and Scope

- **component**: `packages/arkts`
- **purpose**: ArkTS linter integration and diagnostic-provider plumbing used by DevAgent validation flows.
- **primary languages/platforms**: TypeScript, Bun, Node.js, ArkTS tooling

## Important Paths and Interfaces

```text
src/diagnostic-provider.ts   # DiagnosticProvider integration
src/linter.ts                # ArkTS linter subprocess wrapper
src/rules.ts                 # ArkTS rule definitions and filtering
src/index.ts                 # Public exports
src/*.test.ts                # Integration and rule coverage
package.json                 # Build/test scripts
```

## Build, Test, and Run

```bash
cd packages/arkts
bun run build
bun run typecheck
bun run test
```

## Architecture and Workflows

- This package wraps ArkTS lint tooling as a subprocess and adapts it into the runtime diagnostic-provider interface.
- Tests in this component are relatively integration-heavy because they validate subprocess behavior and diagnostics. Preserve that coverage when changing command construction or rule handling.
- Keep the public surface small: callers should consume `createArkTSDiagnosticProvider` through package exports rather than internal files.

## Generated Files, Conventions, and Pitfalls

- Environment-specific lint behavior can make failures look flaky. Prefer changing the narrowest possible code path and verify with the package tests.
- Do not edit `dist/`; it is generated by the build.
- Keep dependency direction one-way: `arkts` can depend on `runtime`, but runtime should not depend back on ArkTS.

## Local Skills

- No component-local skill directory exists here. Use the shared repo skills from `.agents/skills`, especially `testing`, `debug-test-failure`, and `verification-checklist`.
